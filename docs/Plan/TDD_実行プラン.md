## TDD実践ガイド

- 各機能ごとに「テストコード作成→実装→リファクタ→テストパス」を徹底
- テストは`tests/`配下にAPI/サービス/ユーティリティ単位で作成
- カバレッジ80%以上を維持
- Pull Request単位で進捗を管理
